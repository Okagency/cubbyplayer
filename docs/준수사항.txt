[준수사항]
* 코드 수정 시 반드시 암호 "ㄱㄱ" 입력 필요. 절대 예외 없음. 수정 전에 반드시 AskUserQuestion 다이얼로그로 물어볼 것 (선택지에 "ㄱㄱ" 포함).
   그리고 이건 코드 수정할때마다 물어야 함. 기존에 암호를 받았으니 그냥 진행하거나 하지 말 것. 즉 임의로 판단하지 말 것.
* 대화 시작 시 반드시 docs/준수사항.txt를 Read 도구로 열어서 읽고, 읽었음을 피드백할 것 (사용자가 요청하기 전에 먼저 수행)
* 새 대화 시작 후 짧은 단어 입력 → 탭 이름 지정 목적 (작업 요청 아님, 응답 불필요) 첫번째 단어만 응답불필요 하며 두번째부터는 대응해라.
* 에뮬레이터를 실행할 경우는 특별한 지시가 없는 한 앱은 실행하지 말 것
* 진행상황 피드백은 한글로 할 것
5. 수정이 끝난 후 별도지시가 있는 경우가 아니라면 빌드체크하지 않는다.
  만약 '빌드해/빌드체크/빌드' 이런 지시가 있는경우 빌드>설치>실행 까지 진행한다.
6. 혹시 프로그램 수정중에 새로운 지시가 있는 경우 특별한 지시가 없는 한 기존에 작업을 완료 한 후 새로운 지시를 따른다.
7. 수정내용은 수정이력 폴더에 저장한다.
 - 폴더: 수정이력
 - 파일명: yyyyMMdd_HHMM_{수정내용 간략}.txt
   · 시간은 저장 시점의 시스템 시간을 조회하여 사용한다.
   · 예시) 20260324_1531_자주검사 수정.txt
   · 파일명이 이미 존재하면 끝에 (2)를 붙인다.
 - 내용 작성 규칙:
   · 1행: 저장시간 (예: 2026/03/24 15:31)
   · 2행: 빈 줄
   · 3행~: 수정내용 기재 (수정 파일 경로, 변경 내용, 사유)
     롤백까지 가능한 수준으로 수정이력 작성한다.
